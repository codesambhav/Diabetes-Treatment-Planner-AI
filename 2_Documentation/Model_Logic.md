Power BI Model Logic: AI Diabetes PredictionThis document explains the technical architecture and workflow of the Power BI prediction tool.Last Updated: 24-July-20251. Problem Statement & AbstractProblem Statement: Personalized Treatment Planning.Abstract: Develop a machine learning model to create personalized treatment plans for patients, considering their medical history, genetic information, and current health conditions. This solution enables precision medicine, improving treatment effectiveness and patient satisfaction.2. Project ArchitectureThis Power BI project functions as an interactive tool for predicting diabetes status, type, and recommended treatment using machine learning. The entire prediction pipeline is executed within Power Query using a Python script, which dynamically trains models and generates predictions based on user-provided parameters.The dataflow is as follows:User Input (Parameters) ➔ Power Query Table ➔ Python Script (Train & Predict) ➔ Output Table ➔ Power BI Visuals3. Core Logic: Python Script Execution 🐍The core of this project is a Python script embedded within a Power Query step. It does not use pre-trained models; instead, it retrains the models every time the data is refreshed.Input: The script receives a single-row table (PatientInputTable) which is populated by the user via the "Edit parameters" window.Processing Steps:Load Training Data: It loads the historical dataset from C:\code\diabetes_enhanced_data.csv.Train Models: It trains three distinct scikit-learn models on the fly:status_model (Logistic Regression): Predicts if a patient is diabetic.type_model (Random Forest): Predicts the type of diabetes if the patient is diabetic.treatment_model (Random Forest): Recommends a treatment if the patient is diabetic.Predict: It uses the freshly trained models to make a prediction on the single row of patient data passed in from the Power BI parameters.Output: The script returns a single-row table (output_df) containing both the original input parameters and the new prediction columns (Diagnosis, Diagnosis_Confidence, etc.). This table is what feeds the visuals in the report.4. How to Use the Report ⚙️To get a new prediction for a new patient, follow these steps precisely:In the Power BI Desktop ribbon, navigate to Home > Transform data.In the dropdown, click on Edit parameters.The "Edit Parameters" dialog box will appear. Enter the new patient's clinical data into the corresponding fields.Click OK. This updates the PatientInputTable but does not run the model yet.In the Home ribbon, click the main Refresh button. This triggers the Power Query refresh, runs the entire Python script, and generates the new prediction.The visuals on the report canvas (cards and gauges) will now display the results for the new patient.5.
